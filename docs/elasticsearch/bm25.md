# BM25 关键词检索

`ElasticsearchBM25Retriever` 是基于 Elasticsearch BM25 全文检索的 `KeywordRetriever` 实现。使用 `match` 查询对 `content` 字段进行检索，支持可配置的分词器。

## 概述

BM25（Best Matching 25）是一种经典的基于概率的信息检索排序算法，被 Elasticsearch 作为默认的全文检索评分模型。`ElasticsearchBM25Retriever` 将 BM25 检索封装为 `KeywordRetriever` 接口的实现，使其可以与向量检索组合为混合检索。

## 特性

- 基于 Elasticsearch `match` 查询的 BM25 全文检索
- 支持多索引检索
- 可配置分词器（默认 `ik_smart`）
- 可作为混合检索的关键词路径使用

## Builder 配置

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `client` | ElasticsearchClient（必填） | - |
| `indices` | 索引列表（至少指定一个） | - |
| `analyzer` | 检索时使用的分词器 | `ik_smart` |

```java
// 单索引
ElasticsearchBM25Retriever bm25 = ElasticsearchBM25Retriever.builder(esClient)
        .addIndex("knowledge_chunks")
        .build();

// 多索引
ElasticsearchBM25Retriever bm25 = ElasticsearchBM25Retriever.builder(esClient)
        .addIndex("knowledge_chunks")
        .addIndex("other_chunks")
        .build();

// 使用列表设置索引
ElasticsearchBM25Retriever bm25 = ElasticsearchBM25Retriever.builder(esClient)
        .indices(Arrays.asList("knowledge_chunks", "other_chunks"))
        .build();

// 自定义分词器
ElasticsearchBM25Retriever bm25 = ElasticsearchBM25Retriever.builder(esClient)
        .addIndex("knowledge_chunks")
        .analyzer("ik_max_word")
        .build();
```

| Builder 方法 | 说明 |
|-------------|------|
| `addIndex(String)` | 添加一个索引（支持链式调用） |
| `indices(List<String>)` | 设置索引列表 |
| `analyzer(String)` | 设置分词器 |

## API 说明

### 检索

```java
// 基础检索
List<SearchResult> results = bm25.search("搜索关键词", 10);

// 空查询返回空结果
List<SearchResult> empty = bm25.search("", 10);  // 返回空列表
List<SearchResult> empty2 = bm25.search(null, 10);  // 返回空列表
```

| 方法 | 说明 |
|------|------|
| `search(String queryText, int topK)` | 执行 BM25 检索 |

### 搜索结果

`SearchResult` 包含以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `knowledgeBaseId()` | String | 知识库 ID |
| `documentId()` | String | 文档 ID |
| `chunkId()` | String | 文档块 ID |
| `content()` | String | 文档内容 |
| `score()` | double | BM25 评分 |
| `metadata()` | Map | 元数据 |
| `chunkIndex()` | Integer | 文档块序号 |

## 错误处理

- 如果索引不存在，返回空列表（不抛异常）
- 其他 Elasticsearch 错误会包装为 `RuntimeException` 抛出

## 与 ElasticsearchKnowledgeStore 配合使用

推荐通过 `ElasticsearchKnowledgeStore.createBM25Retriever()` 创建 BM25 检索器，确保使用相同的索引和分词器配置：

```java
ElasticsearchKnowledgeStore store = ElasticsearchKnowledgeStore.builder(esClient, 1024)
        .indexName("knowledge_chunks")
        .analyzer("ik_smart")
        .build();

// 自动创建与 Store 同配置的 BM25 检索器
ElasticsearchBM25Retriever bm25 = store.createBM25Retriever();
```

## 完整示例

```java
import co.elastic.clients.elasticsearch.ElasticsearchClient;
import co.elastic.clients.json.jackson.JacksonJsonpMapper;
import co.elastic.clients.transport.rest_client.RestClientTransport;
import com.non.chain.embedding.DashScopeEmbeddingModel;
import com.non.chain.embedding.EmbeddingModel;
import com.non.chain.knowledge.DocumentChunk;
import com.non.chain.knowledge.SearchResult;
import com.non.chain.knowledge.elasticsearch.ElasticsearchBM25Retriever;
import com.non.chain.knowledge.elasticsearch.ElasticsearchKnowledgeStore;
import org.apache.http.HttpHost;
import org.elasticsearch.client.RestClient;

import java.util.Arrays;
import java.util.List;

public class BM25Example {
    public static void main(String[] args) throws Exception {
        // 1. 构建 ES 客户端
        RestClient restClient = RestClient.builder(
                new HttpHost("localhost", 9200)).build();
        RestClientTransport transport = new RestClientTransport(
                restClient, new JacksonJsonpMapper());
        ElasticsearchClient esClient = new ElasticsearchClient(transport);

        // 2. 初始化 Store 并写入数据
        EmbeddingModel embeddingModel = new DashScopeEmbeddingModel("text-embedding-v4");

        ElasticsearchKnowledgeStore store = ElasticsearchKnowledgeStore.builder(esClient, 1024)
                .build();

        String kbId = "kb-bm25-demo";
        String docId = "doc-bm25-001";

        List<String> paragraphs = Arrays.asList(
                "Java 是一种广泛使用的编程语言。",
                "Spring Boot 是 Java 生态中最流行的应用框架。",
                "Elasticsearch 提供强大的全文搜索能力。"
        );

        List<DocumentChunk> chunks = new ArrayList<>();
        for (int i = 0; i < paragraphs.size(); i++) {
            float[] embedding = embeddingModel.embed(paragraphs.get(i));
            chunks.add(DocumentChunk.builder(docId, kbId, paragraphs.get(i))
                    .embedding(embedding)
                    .chunkIndex(i)
                    .build());
        }
        store.addAll(chunks);

        // 3. 创建 BM25 检索器
        ElasticsearchBM25Retriever bm25 = store.createBM25Retriever();

        // 4. 执行关键词检索
        List<SearchResult> results = bm25.search("Java 编程", 5);

        System.out.println("BM25 检索结果:");
        for (SearchResult result : results) {
            System.out.printf("[%.4f] %s%n", result.score(), result.content());
            System.out.println("  chunkId: " + result.chunkId());
            System.out.println("  chunkIndex: " + result.chunkIndex());
        }

        // 5. 清理
        store.deleteByDocumentId(docId);
        restClient.close();
    }
}
```

## 分词器说明

分词器决定了查询文本和文档内容如何被切分为词项。选择合适的分词器对检索效果至关重要：

| 分词器 | 说明 | 适用场景 |
|--------|------|---------|
| `ik_smart` | 粗粒度分词（默认） | 中文通用场景 |
| `ik_max_word` | 细粒度分词 | 需要更高召回率的场景 |
| `standard` | ES 默认分词器 | 英文场景 |
| `whitespace` | 按空白字符分词 | 特殊需求 |

> 注意：`ik_smart` 和 `ik_max_word` 需要安装 [IK Analysis 插件](https://github.com/infinilabs/analysis-ik)。
